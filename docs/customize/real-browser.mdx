## Overview

You can connect the agent to your real Chrome browser instance, allowing it to access your existing browser profile with all your logged-in accounts and settings. This is particularly useful when you want the agent to interact with services where you're already authenticated.

## Connection Methods

There are several ways to connect to your existing browser:

### Method 1: Connect to Running Browser by PID

If you have Chrome running with remote debugging enabled, you can connect directly using its process ID:

```python
from browser_use import Agent, BrowserSession
from langchain_openai import ChatOpenAI
import asyncio

# First, start Chrome with remote debugging:
# /Applications/Google Chrome.app/Contents/MacOS/Google Chrome --remote-debugging-port=9242

# Then connect using the process ID
session = BrowserSession(browser_pid=12345)  # Replace with actual Chrome PID
await session.start()

agent = Agent(
    task="Your task here",
    llm=ChatOpenAI(model='gpt-4o'),
    browser=session,
)

async def main():
    await agent.run()
    # Browser stays open since we connected to existing instance

if __name__ == '__main__':
    asyncio.run(main())
```

### Method 2: Launch with Custom Chrome Executable

To launch a new Chrome instance using your default profile:

```python
from browser_use import Agent, BrowserSession, BrowserProfile
from langchain_openai import ChatOpenAI
import asyncio

# Configure to use your Chrome installation
profile = BrowserProfile(
    # Path to your Chrome executable
    executable_path='/Applications/Google Chrome.app/Contents/MacOS/Google Chrome',  # macOS
    # For Windows: 'C:\\Program Files\\Google\\Chrome\\Application\\chrome.exe'
    # For Linux: '/usr/bin/google-chrome'
    
    # Use your default Chrome profile
    user_data_dir='~/Library/Application Support/Google/Chrome',  # macOS
    # For Windows: '~/AppData/Local/Google/Chrome/User Data'
    # For Linux: '~/.config/google-chrome'
    
    profile_directory='Default',  # or 'Profile 1', 'Profile 2', etc.
    headless=False
)

session = BrowserSession(browser_profile=profile)
await session.start()

agent = Agent(
    task="Your task here",
    llm=ChatOpenAI(model='gpt-4o'),
    browser=session,
)

async def main():
    await agent.run()
    
    input('Press Enter to close the browser...')
    await session.stop()

if __name__ == '__main__':
    asyncio.run(main())
```

### Method 3: Connect via Chrome DevTools Protocol

If Chrome is already running with remote debugging enabled:

```python
from browser_use import Agent, BrowserSession
from langchain_openai import ChatOpenAI
import asyncio

# Connect to Chrome via CDP
session = BrowserSession(cdp_url="http://localhost:9222")
await session.start()

agent = Agent(
    task="Your task here",
    llm=ChatOpenAI(model='gpt-4o'),
    browser=session,
)
```

## Finding Chrome Profile Paths

### Default Profile Locations

- **macOS**: `~/Library/Application Support/Google/Chrome`
- **Windows**: `%LOCALAPPDATA%\Google\Chrome\User Data`
- **Linux**: `~/.config/google-chrome`

### Finding Your Profile Directory

1. Open Chrome and navigate to `chrome://version`
2. Look for "Profile Path" - this shows your current profile location
3. The profile directory name is usually "Default" or "Profile 1", "Profile 2", etc.

## Security Considerations

<Warning>
  When using your real browser profile, the agent will have access to:
  - All your logged-in sessions and cookies
  - Saved passwords (if autofill is enabled)
  - Browser history and bookmarks
  - Extensions and their data
  
  Always review the task you're giving to the agent and ensure it aligns with your security requirements!
</Warning>

## Best Practices

1. **Use a dedicated profile**: Consider creating a separate Chrome profile specifically for agent use:
   ```python
   profile = BrowserProfile(
       user_data_dir='~/.browseruse/profiles/agent-profile',
       profile_directory='Default'
   )
   ```

2. **Limit domain access**: Restrict which sites the agent can visit:
   ```python
   profile = BrowserProfile(
       allowed_domains=['github.com', '*.github.com'],
       # ... other settings
   )
   ```

3. **Enable keep_alive for multiple tasks**: Keep the browser open between agent runs:
   ```python
   profile = BrowserProfile(
       keep_alive=True,
       # ... other settings
   )
   ```

## Troubleshooting

### Chrome Won't Connect

If you're having trouble connecting:

1. **Close all Chrome instances** before trying to launch with a custom profile
2. **Check if Chrome is running with debugging port**:
   ```bash
   ps aux | grep chrome | grep remote-debugging-port
   ```
3. **Verify the executable path** is correct for your system
4. **Check profile permissions** - ensure your user has read/write access

### Profile Lock Issues

If you get a "profile is already in use" error:

1. Close all Chrome instances
2. The profile will automatically be unlocked when BrowserSession starts
3. Alternatively, manually delete the `SingletonLock` file in the profile directory
